Buttons

As a convenience, buttons should only be used for actions, while hyperlinks are to be used for objects. For instance, "Download" should be a button while "recent activity" should be a link.

Button styles can be applied to anything with the .button class applied. However, typically you'll want to apply these to only link and button elements.

Button Classes

There are 3 button classes:

  • Primary button = used for primary actions such as "Add", "Create", "Save" or "View".
  • Secondary button = used as an alternative action of a form such as "Cancel", "Back" or "Close".
  • Alert button = used as buttons that are destruction actions such as "Delete" or "Remove".

Button States

There are 3 button states:

  • Normal = when the button is not touched and unhovered.
  • Hover = when the button is hovered by a mouse, press/click on, and by touch (on a touch screen).
  • Disabled = when the button is disabled so that it cannot be interacted with.

Button Placement

Below screenshots show how to place form buttons based on what page/section. All buttons and links are left-aligned.

Page Actions

Page Actions are placed at the top of the page. Actions dropdown menu button and "Back to previous page" are placed at the top right.


Page Actions on Mobile

There are occasions where the page title may be long. Any button or link beside the title will move down onto the next line.

Form Button Groups

Button Groups are placed at the bottom of the form.

Single Step Forms have at least one of the following button groups:

  • [Save] [Cancel]
  • [Save] [Reset] [Cancel]
  • [Save] [Remove] [Cancel]

Multiple Step Forms have at least one of the following button groups:

  • [Next] [Cancel]
  • [Back] [Next] [Cancel]
  • [Back] [Finish] [Cancel]

Button Groups on Mobile

Buttons are placed the same way on desktop. They also need to be on a single line, if possible.
